**Charting (2026-07-01): Recharts is installed** (`recharts@3`, `apps/web`; network was enabled to add it). Industry-standard interactive panels live in `apps/web/components/charts/rc.tsx` (`"use client"`: `RCArea`/`RCBars`/`RCColumns`/`RCDonut`, themed to `--primary`). GOTCHAS baked in: (1) the shared palette `RC` MUST live in a plain module `components/charts/colors.ts` — a `"use client"` module can't export a constant consumed by a server component (RSC "not found in Client Manifest" 500); server pages import `RC` from `colors`, chart components from `rc`. (2) Every chart is wrapped in a `ChartFrame` mounted-gate (fixed-height placeholder until `useEffect` mount) so SSR HTML matches first client render — prevents hydration mismatch/client exception. (3) All marks set `isAnimationActive={false}` (renders immediately; also fixes blank headless screenshots — Recharts' mount animation never completes under frozen virtual-time). The old dependency-free SVG kit `components/charts/charts.tsx` (`Kpi` etc.) is still used for KPI tiles. To screenshot an AUTH page under snap chromium (CDP blocked), a throwaway PUBLIC route (no leading `_` — `_`-prefixed folders are Next private → 404) that mints a token server-side + renders the panel works with single-shot `--screenshot`; delete + rebuild after.
