# Post-mortems

One file per SEV-1 / SEV-2 incident, named `YYYY-MM-DD-short-slug.md`.

The template and the rules that keep these honest live in
[`../RUNBOOK-INCIDENT-RESPONSE.md`](../RUNBOOK-INCIDENT-RESPONSE.md) §8.

**Before writing a new one, read the previous one's action items.** Repeat
incidents are almost always unfinished actions from last time — and that pattern
is only visible if these sit together in one directory.
